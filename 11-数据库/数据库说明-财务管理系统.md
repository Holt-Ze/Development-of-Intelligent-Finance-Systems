# 数据库说明

本文档用于说明财务管理系统数据库的命名、表结构组成、初始化数据、导入方式和使用注意事项，为系统部署、测试和答辩讲解提供统一依据。

## 1. 数据库名称

建议数据库名称为 `finance_management_system`。

## 2. 数据库脚本内容

当前目录下的 `财务管理系统.sql` 已包含一套可直接演示的初始化脚本，主要内容如下：

- 数据库创建与字符集设置
- 10 张核心业务表的建表语句
- 角色、用户与用户角色关系初始化数据
- 部门、账户、收入类别、支出类别示例数据
- 收入记录与支出记录示例数据
- 备份记录示例数据

## 3. 核心数据表说明

| 表名 | 作用 |
| --- | --- |
| `sys_user` | 系统用户信息 |
| `sys_role` | 角色定义 |
| `sys_user_role` | 用户与角色关联关系 |
| `dept_info` | 部门基础信息 |
| `account_info` | 账户基础信息 |
| `income_category` | 收入类别主数据 |
| `expense_category` | 支出类别主数据 |
| `income_record` | 收入业务记录 |
| `expense_record` | 支出业务记录 |
| `backup_record` | 备份文件记录 |

## 4. 设计特点

- 主键统一使用 `BIGINT AUTO_INCREMENT`
- 主数据表统一提供 `status` 字段，便于启用/停用管理
- 收支记录通过外键关联部门、账户和类别
- 报表统计以 `occurred_on` 和 `amount` 为核心统计字段
- 备份记录单独建表，便于页面追踪历史备份

## 5. 初始化演示数据

脚本默认提供两类演示账号：

- 管理员：`admin / 123456`
- 财务员：`finance01 / 123456`

同时提供若干部门、账户、类别和收支记录，导入后即可直接进行页面演示和报表统计验证。

## 6. 导入方法

1. 启动 MySQL 8 服务。
2. 使用 Navicat、DBeaver、MySQL Workbench 或命令行客户端连接数据库。
3. 执行 `财务管理系统.sql`。
4. 导入完成后，确认 `finance_management_system` 库中的 10 张核心表已创建成功。

命令行示例：

```bash
mysql -u root -p < 11-数据库/财务管理系统.sql
```

## 7. 与系统模块的对应关系

- 登录与权限模块依赖 `sys_user`、`sys_role`、`sys_user_role`
- 基础信息模块依赖 `dept_info`、`account_info`、`income_category`、`expense_category`
- 收入与支出管理模块依赖 `income_record`、`expense_record`
- 备份管理模块依赖 `backup_record`
- 报表统计模块通过查询 `income_record`、`expense_record` 进行聚合分析

## 8. 使用说明与注意事项

- 当前初始化密码为课程演示口径，不适用于生产环境。
- 若修改数据库名称、端口或账号密码，需要同步更新 `10-源代码/backend/src/main/resources/application.yml`。
- 备份记录中的文件路径为示例与运行期结合生成，恢复前应确认备份文件真实存在。
- 若后续扩展预算管理、审批流或审计日志，建议新增独立业务表，不直接挤压现有课程版表结构。
