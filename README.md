# 财务管理系统课程设计

本仓库用于整理《软件工程课程设计》项目的全部交付材料，项目主题为“基于 Web 的学校部门财务管理系统设计与实现”。仓库同时包含课程文档、前后端源码、数据库脚本、测试材料和系统建模图，适合直接用于课程检查、阶段汇报和最终答辩。

## 项目概览

- 项目名称：基于 Web 的学校部门财务管理系统
- 适用场景：学校内部部门收入、支出、账户、类别、报表与备份管理
- 用户角色：管理员、财务员
- 架构方式：前后端分离的 B/S 架构
- 前端技术：Vue 3、Vite、Element Plus、Axios、Vue Router
- 后端技术：Spring Boot 3、MyBatis-Plus、MySQL 8
- 课程目标：覆盖需求分析、系统设计、编码实现、测试验证、文档整理与答辩展示全过程

## 核心功能

- 用户登录、退出与角色区分
- 用户、部门、账户、收入类别、支出类别维护
- 收入记录与支出记录新增、修改、状态管理和查询
- 周报表、月报表、年报表、自定义区间报表和首页仪表盘统计
- 数据备份、下载与恢复

## 运行说明

### 1. 数据库准备

1. 安装并启动 MySQL 8。
2. 执行 `11-数据库/财务管理系统.sql` 初始化数据库和演示数据。
3. 默认数据库名为 `finance_management_system`。

### 2. 启动后端

1. 进入 `10-源代码/backend`。
2. 根据本机环境检查 `src/main/resources/application.yml` 中的数据库连接。
3. 执行 `mvn spring-boot:run`。
4. 默认服务地址为 `http://localhost:8080`。

### 3. 启动前端

1. 进入 `10-源代码/frontend`。
2. 执行 `npm install`。
3. 执行 `npm run dev`。
4. 浏览器访问 Vite 提示的本地地址。

## 答辩建议展示顺序

1. 先介绍项目背景、角色和核心业务流程。
2. 展示需求规格说明书中的功能范围与权限矩阵。
3. 演示系统建模图、数据库设计和总体架构图。
4. 登录系统完成“主数据维护 -> 收支录入 -> 报表统计 -> 数据备份”的主流程。
5. 最后补充测试覆盖情况、项目总结和后续扩展方向。
