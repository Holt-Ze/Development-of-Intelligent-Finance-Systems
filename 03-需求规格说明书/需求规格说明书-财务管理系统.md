# 需求规格说明书

## 1. 引言

### 1.1 编写目的

本文档用于明确学校部门财务管理系统的业务目标、功能需求、非功能需求、角色权限、业务流程和验收依据，为后续设计、实现、测试和答辩提供统一基线。

### 1.2 项目背景

本系统面向学校内部部门的日常财务收支管理场景，主要解决部门收入、支出、账户、类别、报表和备份等业务的信息化管理问题。

### 1.3 术语定义

| 术语 | 说明 |
| --- | --- |
| 管理员 | 负责系统用户、部门、账户、类别和备份管理的角色 |
| 财务员 | 负责收入、支出和报表查看的角色 |
| 部门 | 校内财务归口单位，如学院办公室、学生工作部门等 |
| 账户 | 用于收支记录归类的资金账户 |
| 收入类别 | 收入性质分类，如经费拨入、专项收入 |
| 支出类别 | 支出性质分类，如办公支出、活动支出 |
| 有效状态 | 可参与查询和统计的业务状态 |
| 停用状态 | 主数据保留但不再允许新增业务引用的状态 |

## 2. 总体描述

### 2.1 产品定位

系统是一个基于 B/S 架构的 Web 财务管理系统，采用前后端分离方式建设，部署后可供管理员和财务员通过浏览器访问。

### 2.2 用户特征

- 管理员具备基础信息维护和系统管理能力。
- 财务员具备录入和查询财务业务数据能力。
- 用户对复杂技术细节不敏感，更关注页面简洁、录入方便、统计准确。

### 2.3 运行环境

- 浏览器：Chrome、Edge 等现代浏览器
- 前端：Vue 3 单页应用
- 后端：Spring Boot 应用服务
- 数据库：MySQL 8

### 2.4 设计约束

- 本次仅覆盖课程题目规定功能。
- 不包含预算、审批、报销附件、多组织租户等复杂场景。
- 每条收入或支出记录必须关联一个部门和一个账户。
- 已被业务引用的主数据不得物理删除，只能停用。

## 3. 业务目标

- 统一管理校内部门财务收支信息。
- 提高财务录入与查询效率。
- 保证报表统计口径一致。
- 保留备份记录，降低误操作风险。
- 为课程设计形成完整、清晰、可展示的系统成果。

## 4. 角色与权限

| 功能 | 管理员 | 财务员 |
| --- | --- | --- |
| 登录/退出 | 支持 | 支持 |
| 用户管理 | 支持 | 不支持 |
| 部门管理 | 支持 | 只读 |
| 账户管理 | 支持 | 只读 |
| 收入类别管理 | 支持 | 只读 |
| 支出类别管理 | 支持 | 只读 |
| 收入录入与查询 | 支持 | 支持 |
| 支出录入与查询 | 支持 | 支持 |
| 报表查看 | 支持 | 支持 |
| 数据备份与恢复 | 支持 | 不支持 |

## 5. 功能需求

### 5.1 登录与权限管理

#### 5.1.1 功能描述

用户通过用户名和密码登录系统，系统根据角色返回可访问的菜单和操作权限。

#### 5.1.2 输入

- 用户名
- 密码

#### 5.1.3 输出

- 登录成功信息
- 身份令牌 token
- 用户角色及菜单权限

#### 5.1.4 规则

- 用户名和密码不能为空。
- 停用用户不允许登录。
- 登录失败需给出明确提示。

### 5.2 用户管理

#### 5.2.1 功能点

- 新增用户
- 修改用户信息
- 重置密码
- 停用/启用用户
- 查询用户列表

#### 5.2.2 约束

- 仅管理员可操作。
- 用户名唯一。
- 角色限定为管理员或财务员。

### 5.3 部门管理

#### 5.3.1 功能点

- 新增部门
- 编辑部门名称、负责人、联系电话
- 停用/启用部门
- 查询部门列表

#### 5.3.2 约束

- 若部门已被收支记录引用，则不能删除。
- 停用部门不能再被新记录选择。

### 5.4 账户管理

#### 5.4.1 功能点

- 新增账户
- 编辑账户名称、账号说明、开户单位
- 停用/启用账户
- 查询账户列表

#### 5.4.2 约束

- 已被业务引用的账户只能停用。

### 5.5 收入类别管理

- 新增、编辑、停用收入类别
- 按名称查询收入类别
- 类别被引用后不可物理删除

### 5.6 支出类别管理

- 新增、编辑、停用支出类别
- 按名称查询支出类别
- 类别被引用后不可物理删除

### 5.7 收入信息管理

#### 5.7.1 功能点

- 新增收入记录
- 修改收入记录
- 查看收入记录详情
- 按部门、账户、类别、时间区间查询
- 停用收入记录

#### 5.7.2 字段要求

- 编号
- 所属部门
- 所属账户
- 收入类别
- 金额
- 发生日期
- 经办人
- 备注
- 创建时间
- 状态

#### 5.7.3 规则

- 金额必须大于 0。
- 发生日期不能为空。
- 部门、账户、类别必须为有效状态。

### 5.8 支出信息管理

与收入信息管理一致，字段改为支出类别，支持同类查询和停用规则。

### 5.9 财务统计与报表

#### 5.9.1 报表类型

- 周报表
- 月报表
- 年报表
- 自定义时间区间报表

#### 5.9.2 统计维度

- 按部门
- 按类别
- 按时间趋势

#### 5.9.3 输出内容

- 收入总额
- 支出总额
- 结余
- 类别汇总列表
- 部门汇总列表
- 时间趋势列表

#### 5.9.4 规则

- 仅统计有效状态记录。
- 可按部门过滤。
- 报表首页仪表盘复用汇总结果。

### 5.10 数据备份与恢复

#### 5.10.1 功能点

- 手动创建备份
- 查看备份记录
- 下载备份文件
- 执行恢复操作

#### 5.10.2 规则

- 仅管理员可操作。
- 恢复前需二次确认。
- 恢复记录需留痕。

## 6. 用例说明

### 6.1 用例 UC-01 用户登录

- 参与者：管理员、财务员
- 前置条件：用户账号有效
- 基本流程：
  1. 用户输入用户名和密码。
  2. 系统校验身份。
  3. 系统返回 token、角色和首页信息。
- 异常流程：
  1. 用户名或密码错误。
  2. 账号停用。

### 6.2 用例 UC-02 维护基础信息

- 参与者：管理员
- 前置条件：管理员已登录
- 基本流程：
  1. 进入部门、账户或类别管理页。
  2. 新增或编辑主数据。
  3. 系统校验唯一性和状态。
  4. 保存成功并返回列表。

### 6.3 用例 UC-03 录入收入记录

- 参与者：管理员、财务员
- 前置条件：部门、账户、收入类别均为有效状态
- 基本流程：
  1. 选择部门、账户、收入类别。
  2. 输入金额、日期、经办人和备注。
  3. 系统校验后保存记录。

### 6.4 用例 UC-04 录入支出记录

流程与 UC-03 一致，类别更换为支出类别。

### 6.5 用例 UC-05 查询报表

- 参与者：管理员、财务员
- 基本流程：
  1. 选择统计类型和时间范围。
  2. 可选指定部门。
  3. 系统返回汇总值、分类占比和趋势结果。

### 6.6 用例 UC-06 数据备份与恢复

- 参与者：管理员
- 基本流程：
  1. 管理员手动触发备份。
  2. 系统记录备份信息。
  3. 需要恢复时，管理员选择备份项并确认恢复。

## 7. 业务流程

### 7.1 收支录入流程

1. 管理员预先维护部门、账户和类别。
2. 财务员登录系统。
3. 财务员录入收入或支出数据。
4. 系统校验数据合法性并写入数据库。
5. 报表模块读取有效数据进行统计。

### 7.2 备份流程

1. 管理员进入备份管理页。
2. 手动发起数据库逻辑备份。
3. 系统生成备份文件并记录元数据。
4. 需要恢复时执行二次确认后恢复。

## 8. 非功能需求

### 8.1 可用性

- 界面布局清晰，常用操作在 3 次点击内可完成。
- 表单提示明确，错误信息可读。

### 8.2 性能

- 一般查询响应时间控制在 3 秒内。
- 单次报表查询在课程演示数据量下应平稳返回。

### 8.3 安全性

- 所有接口需要登录后访问。
- 菜单和关键按钮需基于角色控制。
- 密码应加密存储。

### 8.4 可维护性

- 前后端分层清晰。
- 字段命名统一。
- 设计文档与代码目录保持对应关系。

## 9. 验收标准

- 必做功能全部具备清晰设计和可演示入口。
- 角色权限满足题目要求。
- 报表支持周、月、年和自定义区间。
- 已引用的主数据只能停用。
- 备份与恢复流程可说明、可演示、可追踪。
