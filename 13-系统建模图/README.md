# 系统建模图与项目图

本目录集中存放财务管理系统课程设计中的 UML 图、ER 图、数据流图和架构图等建模资料。所有图形均围绕需求规格说明书、设计说明书、数据库脚本和实际源码整理，可直接用于阶段汇报和答辩展示。

## 目录说明

- `dot/`：Graphviz 源文件，可继续维护。
- `svg/`：导出的可编辑 SVG 成品。
- `render.sh`：重新生成全部 SVG 的脚本。

## 建模目标

- 用图形方式说明系统角色、业务流程和模块关系。
- 建立需求、设计、数据库与实现之间的可视化对应关系。
- 为答辩时的架构讲解、数据库讲解和流程演示提供辅助材料。

## 生成策略

- 简单图：使用 `dot` 自动生成，要求仅使用直线或 90° 折线。
- 复杂图：使用规范版式参考后手工维护 SVG，避免 `dot` 自动排版导致乱线。

当前手工维护的复杂图如下：

- `01-use-case.svg`
- `02-class.svg`
- `03-sequence.svg`
- `09-communication.svg`
- `12-er.svg`

## 主要图形用途说明

- 用例图：说明管理员、财务员与系统功能之间的关系。
- 类图：说明实体、业务对象及其关联。
- 时序图：说明关键业务流程中的调用顺序。
- 活动图与流程图：说明收支录入、统计和备份等业务步骤。
- 组件图与部署图：说明前端、后端、数据库的运行关系。
- ER 图：说明数据库表及主外键关系。
- DFD：说明数据在用户、系统和存储之间的流动方式。

## 重新生成

在项目根目录执行：

```bash
bash 13-系统建模图/render.sh
```

说明：

- 该脚本当前只重建简单图，不会覆盖手工维护的复杂图。
- 执行环境需要本机已安装 Graphviz 的 `dot` 命令。
- 当前脚本自动生成的图包括：活动图、状态机图、组件图、部署图、对象图、包图、DFD、流程图和架构图。

## 答辩使用建议

1. 先展示用例图和架构图，快速介绍系统边界与总体结构。
2. 再展示 ER 图和类图，说明数据与对象设计。
3. 如老师追问业务流程，可补充时序图、活动图和 DFD。
4. 若需要说明项目组织过程，可配合甘特图和燃尽图进行讲解。
