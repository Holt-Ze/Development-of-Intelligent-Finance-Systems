# 项目结项总结

## 1. 项目完成情况

本项目围绕学校部门财务管理场景，完成了需求分析、总体设计、数据库设计、接口设计、页面设计、测试分析和课程交付文档整理等工作，形成了一套可用于课程答辩展示的财务管理系统方案。

## 2. 成果概述

- 完成开题报告、计划书、需求规格说明书、软件设计说明书
- 完成测试分析报告、用户手册、会议记录和结项总结
- 完成数据库建表脚本与初始化数据
- 完成前后端工程骨架和页面清单
- 完成答辩提纲和测试用例整理

## 3. 经验总结

### 3.1 成功经验

- 先锁定范围，再展开设计，避免需求蔓延。
- 文档与设计同步推进，减少后期补写压力。
- 统一“部门、账户、类别、收支、报表、备份”的业务主线，便于答辩展示。

### 3.2 存在问题

- 课程周期较短，完整实现所有细节存在时间压力。
- 数据备份和恢复在课程版中以逻辑设计为主，尚未扩展为正式运维方案。
- 扩展功能暂未纳入一期实现。

## 4. 改进方向

- 增加 Word/PDF 报表导出
- 增加预算对比功能
- 增加操作日志和审计功能
- 增加更细粒度权限控制

## 5. 个人收获

- 熟悉了软件工程从需求到设计、测试和总结的完整流程。
- 提升了团队协作、文档规范和任务拆分能力。
- 对 Web 信息管理系统的分层架构和数据库设计有了更系统的理解。

## 6. 结论

本次课程设计选题适中、业务清晰，较好地锻炼了团队在分析、设计、实现组织和文档编写方面的综合能力，为后续进一步实现完整系统提供了扎实基础。
