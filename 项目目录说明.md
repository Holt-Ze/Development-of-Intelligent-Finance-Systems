# 项目目录说明

本目录按《软件工程课程设计》的常见提交流程整理，便于后续编写文档、存放源码和最终归档。

## 目录结构

```text
ClassDesign/
├── 00-课程资料
├── 01-开题报告
├── 02-软件开发计划书
├── 03-需求规格说明书
├── 04-软件设计说明书
├── 05-测试分析报告
├── 06-用户手册
├── 07-会议记录
├── 08-项目结项总结
├── 09-答辩材料
├── 10-源代码
├── 11-数据库
├── 12-测试用例与截图
└── 项目目录说明.md
```

## 各目录用途

- `00-课程资料`：老师下发的教学大纲、题目文档、模板、案例等原始资料。
- `01-开题报告`：开题报告及其后续修改版本。
- `02-软件开发计划书`：进度计划、人员分工、里程碑安排等。
- `03-需求规格说明书`：系统目标、业务流程、功能需求、非功能需求、用例等。
- `04-软件设计说明书`：总体设计、数据库设计、模块设计、接口设计等。
- `05-测试分析报告`：测试计划、测试用例、测试结果、缺陷分析等。
- `06-用户手册`：系统安装说明、操作说明、常见问题等。
- `07-会议记录`：每次小组讨论、任务分配、阶段复盘记录。
- `08-项目结项总结`：项目总结、收获反思、问题与改进建议。
- `09-答辩材料`：答辩 PPT、演示脚本、演示视频或答辩备注。
- `10-源代码`：前端、后端、配置文件、运行说明等源码内容。
- `11-数据库`：数据库设计文档、建表 SQL、初始化数据、备份文件。
- `12-测试用例与截图`：测试截图、运行截图、关键功能演示图片。

- 文档统一使用同一种标题层级和编号规范。

- `10-源代码` 中建议后续再细分为 `frontend`、`backend`、`docs`、`scripts` 等子目录。
