# 项目目录说明

本文档用于说明课程设计仓库的目录组织方式、各类交付物的存放位置以及推荐的维护规则，便于团队协作、阶段检查和最终归档。当前项目主题为“基于 Web 的学校部门财务管理系统”，目录设计遵循“文档、源码、数据库、测试、建模图分层归档”的原则。

## 目录结构

```text
ClassDesign/
├── README.md
├── 01-开题报告
├── 02-软件开发计划书
├── 03-需求规格说明书
├── 04-软件设计说明书
├── 05-测试分析报告
├── 06-用户手册
├── 07-会议记录
├── 08-项目结项总结
├── 09-答辩材料
├── 10-源代码
├── 11-数据库
├── 12-测试用例与截图
├── 13-系统建模图
└── 项目目录说明.md
```

## 各目录用途

- `README.md`：项目总入口，概述项目背景、技术栈、运行方式和主要文档位置。
- `01-开题报告`：开题报告及其后续修改版本。
  当前目录下额外保留了 `00-课程资料` 子目录，用于存放开题阶段参考的课程资料、模板与案例。
- `02-软件开发计划书`：进度计划、人员分工、里程碑安排等。
- `03-需求规格说明书`：系统目标、业务流程、功能需求、非功能需求、用例等。
- `04-软件设计说明书`：总体设计、数据库设计、模块设计、接口设计等。
- `05-测试分析报告`：测试计划、测试用例、测试结果、缺陷分析等。
- `06-用户手册`：系统安装说明、操作说明、常见问题等。
- `07-会议记录`：每次小组讨论、任务分配、阶段复盘记录。
- `08-项目结项总结`：项目总结、收获反思、问题与改进建议。
- `09-答辩材料`：答辩 PPT、演示脚本、演示视频或答辩备注。
- `10-源代码`：前端、后端、配置文件、运行说明等源码内容。
- `11-数据库`：数据库设计文档、建表 SQL、初始化数据、备份文件。
- `12-测试用例与截图`：测试截图、运行截图、关键功能演示图片。
- `13-系统建模图`：UML 图、ER 图、DFD、架构图等建模资料及导出脚本。

## 交付物之间的关系

- `01-开题报告` 负责说明选题背景、研究意义和目标，是立项依据。
- `02-软件开发计划书` 负责定义进度、分工、里程碑和风险，是执行依据。
- `03-需求规格说明书` 明确系统要做什么，是设计与测试的需求基线。
- `04-软件设计说明书` 说明系统如何实现，是编码和数据库设计的实现依据。
- `05-测试分析报告` 与 `12-测试用例与截图` 一起证明系统被验证过。
- `06-用户手册` 面向演示和使用者，说明如何安装、登录和完成业务操作。
- `08-项目结项总结` 和 `09-答辩材料` 用于课程验收、答辩陈述和复盘总结。
- `10-源代码`、`11-数据库`、`13-系统建模图` 为文档提供实现与图形支撑。

## 维护建议

- 文档统一使用同一种标题层级和编号规范。
- 每次完成一个阶段后，及时把文档和截图归档，避免最后集中补材料。
- 重要文档建议在首页、对应目录 README 和答辩提纲中保持名称一致，减少查找成本。
- `10-源代码`、`11-数据库`、`13-系统建模图` 的说明文件应随实际内容同步更新，避免文档与实现脱节。
- 归档前建议统一检查目录中是否残留临时文件、重复截图或无关构建产物。
