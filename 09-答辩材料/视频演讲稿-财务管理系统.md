# 财务管理系统课程设计视频演讲稿

## 1. 使用说明

本讲稿适用于录制课程设计运行演示视频，建议总时长控制在 6 到 8 分钟。  
录制时可以按“画面操作”和“讲解词”同步进行，如果现场操作速度较慢，可适当删减部分描述。

## 2. 录制前准备

- 启动 MySQL，并导入 `11-数据库/财务管理系统.sql`
- 启动后端服务：`mvn spring-boot:run`
- 启动前端服务：`npm install` 后执行 `npm run dev`
- 浏览器打开前端页面
- 准备两个演示账号：
  - 管理员：`admin / 123456`
  - 财务员：`finance01 / 123456`

## 3. 视频开场讲解

### 3.1 开场白

大家好，我演示的课程设计项目是《基于 Web 的学校部门财务管理系统设计与实现》。  
这个系统面向学校内部部门的日常财务业务，主要解决收入、支出、账户、类别、报表统计以及数据备份等管理问题。  
整个系统采用前后端分离的 B/S 架构，前端使用 Vue 3 和 Element Plus，后端使用 Spring Boot 3，数据库使用 MySQL 8。

### 3.2 演示目标

下面我将从系统运行环境开始，依次演示登录、基础数据维护、收支录入、报表统计以及数据备份恢复几个核心流程，展示本系统已经完成的主要功能。

## 4. 运行环境展示

### 画面操作

- 先展示数据库已导入
- 展示后端启动终端
- 展示前端启动终端
- 切换到浏览器首页

### 讲解词

首先我展示系统的运行环境。  
本项目的数据存储在 MySQL 数据库中，数据库名称为 `finance_management_system`，初始化脚本已经提前导入。  
后端服务运行在 `8080` 端口，负责登录认证、主数据管理、收支记录、报表统计和备份接口。  
前端基于 Vue 3 和 Vite 启动，负责页面展示和业务交互。  
当前前后端已经成功连接，所以接下来可以直接演示完整业务流程。

## 5. 系统登录演示

### 画面操作

- 打开登录页
- 输入管理员账号 `admin`
- 点击登录

### 讲解词

现在看到的是系统登录页。  
系统提供两类角色，分别是管理员和财务员，不同角色拥有不同的菜单权限。  
我先使用管理员账号登录。  
登录成功后，系统进入首页仪表盘，可以看到收入总额、支出总额和结余等统计信息，同时右上角也会显示当前登录角色。

## 6. 首页和总体功能介绍

### 画面操作

- 停留在首页仪表盘
- 简单展示左侧菜单

### 讲解词

首页仪表盘主要用于展示系统的总体财务情况，包括收入、支出和结余。  
左侧菜单展示了系统的主要模块，包括用户管理、部门管理、账户管理、收入类别管理、支出类别管理、收入记录、支出记录、报表中心和备份管理。  
其中，用户、部门、账户、类别和备份管理主要由管理员使用，收入、支出和报表模块则管理员和财务员都可以访问。

## 7. 基础数据维护演示

### 7.1 用户管理

#### 画面操作

- 点击“用户管理”
- 展示已有用户
- 如方便可演示新增一条用户数据

#### 讲解词

首先是用户管理模块。  
管理员可以在这里维护系统用户，包括用户名、姓名、电话和角色信息。  
系统通过角色区分管理员和财务员权限，保证不同岗位只能访问对应功能。

### 7.2 部门管理

#### 画面操作

- 点击“部门管理”
- 展示部门列表

#### 讲解词

接下来是部门管理模块。  
这里维护学校内部各个部门的基础信息，后续所有收入和支出记录都需要关联具体部门。  
为了保证历史数据的完整性，当部门已经被业务记录引用时，系统采用停用策略，而不是直接删除。

### 7.3 账户和类别管理

#### 画面操作

- 依次点击“账户管理”“收入类别管理”“支出类别管理”

#### 讲解词

账户管理用于维护财务账户信息，比如账户编号、账户名称和所属单位。  
收入类别和支出类别管理用于维护业务分类，例如经费拨款、活动支出、办公支出等。  
这些主数据为后续业务录入提供统一口径，也为报表统计打下基础。

## 8. 财务业务录入演示

### 8.1 切换财务员登录

#### 画面操作

- 退出当前管理员账号
- 使用 `finance01 / 123456` 登录

#### 讲解词

接下来我切换为财务员账号进行业务录入演示。  
这样可以更清楚地体现系统的角色权限控制。  
登录后可以看到，财务员的菜单明显减少，主要只保留首页仪表盘、收入记录、支出记录和报表中心等业务相关功能。

### 8.2 收入录入

#### 画面操作

- 点击“收入记录”
- 点击新增
- 选择部门、账户、收入类别
- 填写金额、日期、经办人、备注
- 保存

#### 讲解词

这里是收入记录模块。  
新增收入时，系统要求选择部门、账户和收入类别，并填写金额、发生日期和经办人等信息。  
这样可以保证每条收入记录都有完整的业务归属。  
保存成功后，数据会进入收入记录列表，后续也可以参与报表统计。

### 8.3 支出录入

#### 画面操作

- 点击“支出记录”
- 新增一条支出
- 保存

#### 讲解词

支出记录模块与收入记录类似，只是对应的业务类别变为支出类别。  
通过收入和支出两类记录，系统可以形成完整的财务收支台账。  
这部分功能也是本系统日常使用频率最高的核心模块之一。

## 9. 报表统计演示

### 画面操作

- 点击“报表中心”
- 先展示默认月报表
- 切换周报表或年报表
- 再切换到“自定义区间”
- 可选择某个部门进行筛选

### 讲解词

下面演示报表中心。  
这个模块支持周报表、月报表、年报表和自定义区间统计，既可以查看总体财务情况，也可以按部门筛选。  
页面上方可以设置报表类型和部门条件，下方会展示收入总额、支出总额和结余。  
同时系统还会给出类别汇总、部门汇总和趋势汇总，帮助使用者从多个维度分析财务数据。  
这也是本项目相对有亮点的部分，因为它不仅能录入数据，还能对数据进行汇总分析。

## 10. 备份管理演示

### 画面操作

- 退出财务员账号
- 重新使用管理员账号登录
- 点击“备份管理”
- 点击“创建备份”
- 展示备份记录
- 如条件允许，演示下载或恢复确认框

### 讲解词

最后演示备份管理模块。  
这个模块仅管理员可见，用于保障系统数据安全。  
管理员可以手动创建备份，系统会生成备份文件并记录创建时间、创建人、文件大小等信息。  
如果后续数据出现异常，也可以基于已有备份执行恢复操作。  
恢复操作设置了二次确认，目的是防止误操作覆盖当前数据库数据。

## 11. 项目总结讲解

### 讲解词

通过刚才的演示可以看到，本系统已经实现了课程设计要求中的主要业务流程，包括登录认证、角色权限控制、基础数据维护、收支录入、报表统计以及数据备份恢复。  
在技术实现上，系统采用前后端分离架构，页面与接口解耦，便于后续扩展和维护。  
在数据库设计上，共设计了 10 张核心业务表，能够支撑用户、角色、部门、账户、类别、收支记录和备份记录等主要业务对象。  
在业务规则上，系统采用主数据停用策略，避免已被引用的数据被直接删除，从而保护历史记录的一致性。

## 12. 不足与后续改进

### 讲解词

当然，这个系统目前仍然是课程设计版本，后续还有进一步完善空间。  
例如可以增加预算控制、审批流程、导出 Excel 或 PDF、操作日志审计，以及更细粒度的数据权限控制。  
如果继续迭代，本项目可以从课程设计原型进一步发展为更加完整的财务业务系统。
