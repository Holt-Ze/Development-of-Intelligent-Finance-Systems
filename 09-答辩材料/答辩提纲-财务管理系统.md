# 答辩提纲

## 1. 项目简介

- 项目名称：基于 Web 的学校部门财务管理系统
- 选题来源：课程设计题目第 5 题
- 项目目标：解决校内部门收入、支出、报表和备份管理问题

## 2. 为什么选择这个题目

- 业务边界清晰
- 适合课程设计周期
- 能覆盖需求、设计、数据库、接口、测试等完整流程

## 3. 需求分析

- 角色：管理员、财务员
- 核心模块：登录、基础信息、收入、支出、报表、备份
- 业务约束：每条记录必须关联部门和账户，主数据被引用后只能停用

## 4. 设计方案

- 架构：Vue + Spring Boot + MySQL
- 风格：前后端分离，REST 接口
- 数据表：10 张核心表
- 页面：登录页、仪表盘、主数据管理、收支管理、报表中心、备份管理

## 5. 系统亮点

- 报表支持周、月、年和自定义区间
- 支持部门维度统计
- 采用停用策略保护历史数据
- 预留 Word/PDF 导出扩展能力

## 6. 测试与验证

- 覆盖登录、权限、收支录入、报表统计和备份流程
- 重点验证金额校验、角色权限、主数据引用规则

## 7. 不足与扩展

- 当前以课程版原型和骨架为主
- 未加入预算控制和审批流
- 二期可增加导出、审计和预算功能

## 8. 结束语

- 本项目完成了课程设计要求的主要分析和设计工作
- 如果时间继续推进，可基于现有骨架完成更完整实现
